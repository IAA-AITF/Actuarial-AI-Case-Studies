Stage 1: Extracting Features with LLM

We use a dataset of 3,000 workers’ compensation claims from a Kaggle competition on actuarial loss estimation. The dataset is fully synthetic (stratified sample from 90,000 records) and contains both structured features and free-text claim descriptions.

LLM extraction (pre-computed): OpenAI gpt-4o-mini (model string: gpt-4o-mini, version identifier: gpt-4o-mini-2024-07-18) extracted three fields from each ClaimDescription via the structured-output endpoint:

  • number_of_body_parts_injured — integer count
  • main_body_part_injured — primary anatomical site (e.g., LOWER BACK)
  • cause_of_injury — primary action verb (e.g., fall, strain)

Outputs are cached in ClaimsdatawithPrompts.csv.

from pydantic import BaseModel

system_prompt_adjusted = """
    Your task is to extract structured information about
    injuries and cause of injury from the given text.
    Follow this schema strictly:
    - number_of_body_parts_injured: The total count of injured body parts.
    - main_body_part_injured: The primary body part affected, described
      concisely (e.g., 'HEAD', 'THUMB').
    - cause_of_injury: Specify by verb.
        - If verb given: return only the primary action verb that directly
          caused the injury (e.g. 'fall' not 'fell from box')
        - If cause is not mentioned, infer from context if possible,
          otherwise return 'unspecified'.
    Ensure accuracy and consistency in the extracted details.
    Do not add interpretations beyond the provided text.
"""

class AdditionalInformation(BaseModel):
    number_of_body_parts_injured: int
    main_body_part_injured: str
    cause_of_injury: str

def extract_additional_information(text: str):
    completion = client.beta.chat.completions.parse(
        model=MODEL,
        messages=[
            {"role": "system", "content": system_prompt_adjusted},
            {"role": "user",   "content": text}
        ],
        response_format=AdditionalInformation,
    )
    p = completion.choices[0].message.parsed
    return p.number_of_body_parts_injured, p.main_body_part_injured, p.cause_of_injury

Stage 2: Data Preprocessing

We load the pre-computed dataset, log-transform the target, impute two missing MaritalStatus values, one-hot-encode demographic categorical columns, and engineer date features.

Leakage assessment for all pre-split operations:

Operation Parameters learned from data Applied before split Leakage risk
Log-transform of target None Yes None
MaritalStatus imputation ('S') Fixed label, 2 rows Yes Negligible
pd.get_dummies on Gender / MaritalStatus / PartTimeFullTime None — fixed binary schema Yes None
Date feature engineering None Yes None
Regex category mapping (Stage 3) None — deterministic rules Yes None
pd.get_dummies on body_part_category / cause_of_injury_category (Stage 3) None — fixed closed category set with OTHER catch-all Yes None

Why pd.get_dummies on LLM categories is safe before the split: The regex codebook defines a fixed, closed set of 8 body-part and 13 cause categories. An OTHER catch-all ensures every possible input string maps to one of these predefined labels. No category can appear in test that is absent from train; the complete column structure is therefore fully known before any split. This is the same reasoning that makes it safe to apply pd.get_dummies to Gender and MaritalStatus before the split.

Stage 3: Categorising and Encoding LLM-Extracted Content

Raw LLM outputs contain ~175–224 unique values. We reduce these to 8 anatomical regions and 13 cause categories via deterministic regex rules, then immediately apply pd.get_dummies.

Group Categories
Body part (8) HEAD_FACE, EYES, TORSO, UPPER_EXTREMITY, HAND_FINGERS, LOWER_EXTREMITY, NECK, OTHER
Cause (13) LIFTING_CARRYING, IMPACT, FALL_SLIP_TRIP, STRAIN_SPRAIN, LACERATION, CHEMICAL_BURN, COMPRESSION, CONTACT, ERGONOMIC, WALKING, DRIVING, MEDICAL, OTHER

Why pd.get_dummies is applied here, before the split:

The two key properties that make this safe are:

  1. Closed category set. The codebook hard-codes exactly 8 body-part and 13 cause labels. No new label can ever be produced — the set is defined by the regex rules, not learned from the data.
  2. OTHER is a universal catch-all. Any input string that does not match a specific regex pattern falls into OTHER. Consequently, every row in val and test will always map to one of the predefined categories, and the resulting dummy columns are guaranteed to be identical in structure to those from the training rows.

This reasoning is identical to why pd.get_dummies on Gender and MaritalStatus was safe in Stage 2: the category set is fixed and fully known before any split.

def make_splits(df, target_col=TARGET):
    """
    Applies the globally defined split indices to df.
    reset_index(drop=True) gives a clean 0-based index required for
    iloc-based CV splitting in Steps 2/3/7 and alignment with train_bins.
    """
    X = df.drop(columns=[target_col])
    y = df[target_col]
    return {
        'X_train': X.iloc[train_idx].reset_index(drop=True),
        'X_test' : X.iloc[test_idx].reset_index(drop=True),
        'y_train': y.iloc[train_idx].reset_index(drop=True),
        'y_test' : y.iloc[test_idx].reset_index(drop=True),
    }

Step 2 — Hyperparameter selection via Stratified 4-fold CV¶

Grid search evaluated on the training set using Stratified 4-fold CV. The test set is not touched. Reused for every ablation variant in Step 7 (verbose=False).

Grid: n_estimators ∈ {50, 100, 200} × max_depth ∈ {3, 5, 7} (9 combinations per model).

In [17]:
PARAM_GRID = {'n_estimators': [50, 100, 200], 'max_depth': [3, 5, 7]}

N_CV_SPLITS = 4


def select_hyperparams_cv(splits, strat_bins_train, verbose=True):
    """Grid-search via Stratified 4-fold CV on the training set. Returns best params dict."""
    skf = StratifiedKFold(n_splits=N_CV_SPLITS, shuffle=True, random_state=RANDOM_STATE)
    rows = []
    for n_est in PARAM_GRID['n_estimators']:
        for depth in PARAM_GRID['max_depth']:
            fold_rmses = []
            for fold_tr, fold_va in skf.split(splits['X_train'], strat_bins_train):
                m = GradientBoostingRegressor(
                    n_estimators=n_est, max_depth=depth,
                    learning_rate=0.1, random_state=RANDOM_STATE
                )
                m.fit(splits['X_train'].iloc[fold_tr], splits['y_train'].iloc[fold_tr])
                preds = m.predict(splits['X_train'].iloc[fold_va])
                fold_rmses.append(np.sqrt(mean_squared_error(
                    splits['y_train'].iloc[fold_va], preds)))
            rows.append({
                'n_estimators': n_est, 'max_depth': depth,
                'cv_RMSE_mean': np.mean(fold_rmses),
                'cv_RMSE_std' : np.std(fold_rmses),
            })

Step 3 — Uncertainty quantification: Stratified 4-fold CV¶

Stratified 4-fold CV on the training set using each model's best HPs from Step 2.

Key design choices:

  • StratifiedKFold uses EVAL_RANDOM_STATE (not RANDOM_STATE), so the evaluation folds are independent of the folds used for hyperparameter selection in Step 2. This avoids optimistic bias from evaluating on the same folds that informed HP selection.
  • Both models iterate over identical fold splits — required for paired tests in Step 4.
  • train_bins is reset_index(drop=True), aligning with splits_bl['X_train'].
  • The test set is not accessed here.

def run_paired_cv(splits_bl, splits_eh,
                  best_params_bl, best_params_eh,
                  strat_bins_train, n_splits=N_CV_SPLITS):
    """Stratified K-fold with IDENTICAL folds for both models.

    Uses EVAL_RANDOM_STATE so that evaluation folds are independent
    of the folds used for hyperparameter selection in Step 2.
    """
    skf    = StratifiedKFold(n_splits=n_splits, shuffle=True,
                             random_state=EVAL_RANDOM_STATE)
    X_tr_bl = splits_bl['X_train']
    X_tr_eh = splits_eh['X_train']
    y_tr    = splits_bl['y_train']

    fold_m = {m: {'bl': [], 'eh': []} for m in ('RMSE', 'MAE', 'R2')}

    for fold_tr, fold_va in skf.split(X_tr_bl, strat_bins_train):
        for tag, X_tr, best_p in [('bl', X_tr_bl, best_params_bl),
                                   ('eh', X_tr_eh, best_params_eh)]:
            model = GradientBoostingRegressor(
                **best_p, learning_rate=0.1, random_state=RANDOM_STATE)
            model.fit(X_tr.iloc[fold_tr], y_tr.iloc[fold_tr])
            preds  = model.predict(X_tr.iloc[fold_va])
            y_true = y_tr.iloc[fold_va]
            fold_m['RMSE'][tag].append(np.sqrt(mean_squared_error(y_true, preds)))
            fold_m['MAE'][tag].append(mean_absolute_error(y_true, preds))
            fold_m['R2'][tag].append(r2_score(y_true, preds))

    for m in fold_m:
        for tag in ('bl', 'eh'):
            fold_m[m][tag] = np.array(fold_m[m][tag])
    return fold_m

Step 4 — Paired statistical test on fold-level errors¶

We use the Nadeau & Bengio (2003) corrected paired $t$-test, which accounts for the non-independence of cross-validation fold scores caused by overlapping training sets. The correction inflates the standard error by a factor of $\sqrt{1/K + n_\text{test}/n_\text{train}}$, where $K$ is the number of folds and $n_\text{test}/n_\text{train} = 1/(K-1)$ for $K$-fold CV.

A result is described as statistically significant when the test rejects at $\alpha = 0.05$.

In [19]:
def nadeau_bengio_corrected_ttest(scores_a, scores_b):
    """
    Nadeau & Bengio (2003) corrected paired t-test for K-fold CV.

    Accounts for the non-independence of fold scores due to overlapping
    training sets.  The correction factor is (1/K + n_test/n_train).
    For K-fold CV without repeats: n_test/n_train = 1/(K-1).
    """
    diffs = scores_a - scores_b
    K     = len(diffs)
    d_bar = diffs.mean()
    s2    = diffs.var(ddof=1)

    # Correction: 1/K + 1/(K-1) for K-fold CV
    corrected_var = (1/K + 1/(K - 1)) * s2
    t_stat = d_bar / np.sqrt(corrected_var)
    p_val  = 2 * stats.t.sf(abs(t_stat), df=K - 1)
    return t_stat, p_val


def paired_tests(cv_metrics):
    for metric in ('RMSE', 'MAE'):
        bl = cv_metrics[metric]['bl']
        eh = cv_metrics[metric]['eh']
        t_stat, p_val = nadeau_bengio_corrected_ttest(bl, eh)
        diffs = bl - eh
        sig = p_val < 0.05
        verdict = ('Statistically significant at alpha=0.05'
                   if sig else
                   'Not significant at alpha=0.05')
        print(f'\n── {metric} ──')
        print(f'  Mean reduction              : {diffs.mean():.4f}')
        print(f'  Corrected paired t-test (NB): t={t_stat:+.3f}, p={p_val:.4e}')
        print(f'  Verdict                     : {verdict}')

def stratified_and_quantile_eval(model_bl, model_eh,
                                 splits_bl, splits_eh, df_all):

    preds_eh = model_eh.predict(splits_eh['X_test'])

    # Keep as Series (no .values) so pd.cut returns a Series with .cat accessor

        orig,

def build_summary_table(cv_metrics, test_metrics_bl, test_metrics_eh):
    rows = []
    for key, label, direction in [
        ('RMSE', 'RMSE  (lower is better)',  'lower'),
        ('MAE',  'MAE   (lower is better)',  'lower'),
        ('R2',   'R2    (higher is better)', 'higher'),
    ]:
        bl_cv, eh_cv = cv_metrics[key]['bl'], cv_metrics[key]['eh']
        bl_t,  eh_t  = test_metrics_bl[key], test_metrics_eh[key]
        cv_impr   = (100*(bl_cv.mean()-eh_cv.mean())/bl_cv.mean()
                     if direction=='lower'
                     else 100*(eh_cv.mean()-bl_cv.mean())/abs(bl_cv.mean()))
        test_impr = (100*(bl_t-eh_t)/bl_t
                     if direction=='lower'
                     else 100*(eh_t-bl_t)/abs(bl_t))
        rows.append({
            'Metric'                  : label,
            'Baseline CV (mean+/-SD)' : f'{bl_cv.mean():.4f} +/- {bl_cv.std():.4f}',
            'Enhanced CV (mean+/-SD)' : f'{eh_cv.mean():.4f} +/- {eh_cv.std():.4f}',
            'CV Improvement (%)'      : f'{cv_impr:.1f}%',
            'Baseline Test'           : f'{bl_t:.4f}',
            'Enhanced Test'           : f'{eh_t:.4f}',
            'Test Improvement (%)'    : f'{test_impr:.1f}%',
        })
    return pd.DataFrame(rows)
